# داده آزمایشی کاربران، پروژه‌ها و نمودارها

این بخش برای محیط توسعه ساخته شده است تا بتوان داده کافی برای تست کاربران، پروژه‌ها، فازها، داشبورد مدیریتی و نمای کلان پروژه‌ها ایجاد کرد.

## مسیرها

```http
GET /api/v1/fixtures/users-projects
```

پیش‌نمایش داده‌های آزمایشی را بدون نوشتن در دیتابیس برمی‌گرداند.

```http
POST /api/v1/fixtures/users-projects
Content-Type: application/json

{
  "reset": false
}
```

داده‌های آزمایشی را در دیتابیس ایجاد یا به‌روزرسانی می‌کند.

## داده‌های ساخته‌شده

- ۱۰ کاربر با پیشوند `fixture.`
- ۹ نقش پروژه برای نمایش بار کاری نقش‌ها
- ۱۰ پروژه با وضعیت‌ها و اولویت‌های متنوع
- ۲۰ فاز پروژه، دو فاز برای هر پروژه
- ۱۴ وظیفه در وضعیت‌های مختلف
- ۸ گزارش کار در بازه ۷ روز اخیر
- ۱۳ فایل پروژه و گزارش، شامل فایل‌های دسته `reports`

## پوشش نمودارها

داده‌ها طوری تنظیم شده‌اند که نمودارهای زیر خالی نباشند:

- نمودار وضعیت پروژه‌ها
- Bundle Size Treemap وضعیت پروژه‌ها
- نمودار دایره‌ای مرحله/وضعیت پروژه‌ها
- نمودار اولویت پروژه‌ها
- نمودار وضعیت وظایف
- نمودار عقب‌افتادگی بر اساس نقش
- نمودار عقب‌افتادگی بر اساس کارشناس
- نمودار حجم گزارش‌ها بر اساس پروژه
- نمودار روند ۷ روزه گزارش‌های کاری و وظایف تکمیل‌شده در داشبورد
- لیست آخرین رویدادها شامل پروژه، وظیفه، گزارش و فایل

## رفتار تاریخ‌ها

تاریخ‌های اصلی seed نسبت به روز اجرای seed ساخته می‌شوند. بنابراین همیشه داده‌های زیر وجود دارد:

- پروژه عقب‌افتاده
- پروژه نزدیک سررسید
- پروژه سالم در جریان
- پروژه تکمیل‌شده
- پروژه لغوشده
- وظیفه عقب‌افتاده
- وظیفه امروز
- گزارش‌های کاری در ۷ روز اخیر

## رفتار seed

- اجرای دوباره مسیر Duplicate ایجاد نمی‌کند.
- پروژه‌ها، فازها، وظایف، گزارش‌ها و فایل‌های fixture به‌روزرسانی و بازسازی می‌شوند.
- اگر `reset: true` ارسال شود، داده‌های fixture قبلی حذف و دوباره ساخته می‌شود.

## رمز عبور کاربران آزمایشی

```text
Fixture123456!
```

## امنیت

این مسیرها در production به‌صورت پیش‌فرض بسته هستند. اگر عمداً نیاز به فعال‌سازی در production باشد، باید متغیر زیر تنظیم شود:

```env
ENABLE_FIXTURE_ROUTES=true
```

استفاده در production توصیه نمی‌شود.
