# ماژول ربات تلگرام

## هدف

ربات تلگرام یک رابط عملیاتی برای مشاهده وضعیت پروژه‌ها و ثبت اطلاعات روزانه است. اطلاعات نیروی انسانی از Excel دریافت نمی‌شود؛ بنابراین ربات نیز افراد را از فایل Excel یا نام کاربری آزاد دریافت نمی‌کند و فقط از تخصیص‌های معتبر ثبت‌شده داخل پروژه استفاده می‌کند.

## قابلیت‌های اصلی

- نمایش خلاصه مدیریتی پروژه‌ها، وظایف باز، تأخیرها و گزارش‌های اخیر
- نمایش پروژه‌های قابل دسترسی بر اساس نقش و تخصیص کاربر
- نمایش جزئیات پروژه، فازها، وظایف، گزارش‌ها و جمع مالی فازها
- نمایش پروژه‌های فعال با مسئول، عضو یا نقش ناقص
- ثبت گزارش متنی برای پروژه
- دریافت فایل، سند، تصویر، ویدئو، ویس و فایل صوتی برای گزارش
- تبدیل فایل صوتی به متن و اتصال آن به گزارش
- استخراج درصد پیشرفت از متن‌هایی مانند `پیشرفت 65%` یا `پیشرفت ۶۵٪`
- ثبت وظیفه توسط مدیر برای مسئول یا عضو واقعی پروژه
- انتخاب اولویت و مهلت وظیفه در ربات
- دریافت پیوست وظیفه و تبدیل صوت به متن
- مشاهده جزئیات وظیفه و تغییر وضعیت به «برای انجام»، «در حال انجام»، «مسدود» یا «انجام‌شده»
- دسترسی فقط خواندنی برای نقش هیئت مدیره

## منطق دسترسی پروژه

- `manager`: مشاهده همه پروژه‌ها و ایجاد وظیفه
- `board`: مشاهده همه پروژه‌ها بدون ثبت یا تغییر داده
- `expert`: مشاهده پروژه‌هایی که مالک، عضو، عضو دارای نقش یا مسئول فاز آن‌ها است

## تغییر مهم در تخصیص وظیفه

در نسخه قبلی ربات فقط مدیران را به‌عنوان مسئول وظیفه نمایش می‌داد و در نبود مدیر پروژه به فهرست عمومی مدیران برمی‌گشت. این رفتار حذف شد.

در نسخه جدید:

1. مدیر پروژه را انتخاب می‌کند.
2. ربات مالک، اعضای مستقیم و اعضای دارای نقش همان پروژه را resolve می‌کند.
3. فقط کاربران فعال همان پروژه قابل انتخاب هستند.
4. اگر پروژه هنوز نیروی انسانی ندارد، ثبت وظیفه متوقف می‌شود و مدیر برای تکمیل تخصیص به پنل هدایت می‌شود.

این طراحی از ایجاد وظیفه برای فرد اشتباه جلوگیری می‌کند و با فرآیند جدید Excel سازگار است.

## دستورات

| دستور | عملکرد |

| `/start` | راهنما و منوی اصلی |

| `/task` | ثبت وظیفه جدید |

## API مدیریتی

- `GET /api/v1/telegram/status`
- `GET /api/v1/telegram/overview`
- `GET /api/v1/telegram/users`
- `POST /api/v1/telegram/webhook/configure`
- `DELETE /api/v1/telegram/webhook/configure`
- `POST /api/v1/telegram/test-message`
- `DELETE /api/v1/telegram/users/:userId/link`
- `POST /api/v1/telegram/webhook`

عملیات تنظیم وبهوک، ارسال پیام آزمایشی و حذف اتصال کاربر فقط برای مدیر مجاز است.

## متغیرهای محیطی

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_WEBHOOK_SECRET`
- `TELEGRAM_BOT_PUBLIC_URL`
- `TELEGRAM_CHANNEL_ID`
- `TELEGRAM_DAILY_ALERT_ENABLED`
- `TELEGRAM_DAILY_ALERT_TIME`
- `TELEGRAM_DAILY_ALERT_TIMEZONE`
- `OPENAI_API_KEY`
- `OPENAI_TRANSCRIPTION_MODEL`
- `OPENAI_TRANSCRIPTION_LANGUAGE`

`TELEGRAM_BOT_PUBLIC_URL` باید HTTPS باشد. Backend مسیر `/api/v1/telegram/webhook` را به آن اضافه می‌کند و Secret Token را از طریق Header استاندارد Telegram اعتبارسنجی می‌کند.

## sessionها

- `TelegramBotSession`: فرآیند ثبت گزارش و پیوست
- `TelegramTaskSession`: فرآیند ثبت وظیفه

session وظیفه در MongoDB ذخیره می‌شود و بعد از ۲۴ ساعت به‌صورت TTL حذف می‌شود. workflow قدیمی in-memory حذف و به سرویس واحد task bot تبدیل شده است.


## اتصال امن کاربر بدون ثبت دستی شناسه

اتصال کاربران با کد یک‌بارمصرف انجام می‌شود:

1. مدیر از صفحه مدیریت ربات برای کاربر کد اتصال می‌سازد.
2. کد ۸ کاراکتری فقط ۱۵ دقیقه معتبر است و یک‌بار مصرف می‌شود.
3. کاربر دستور `/link CODE` را برای ربات ارسال می‌کند یا Deep Link ساخته‌شده در پنل را باز می‌کند.
4. ربات `Telegram User ID`، `Chat ID` و Username را مستقیماً از Update معتبر تلگرام ذخیره می‌کند.
5. یک حساب تلگرام نمی‌تواند همزمان به دو کاربر پنل متصل باشد.

این روش جایگزین ورود دستی شناسه‌هاست و خطای انتساب حساب را کاهش می‌دهد.

API مدیر:

- `POST /api/v1/telegram/users/:userId/link-code`
- `DELETE /api/v1/telegram/users/:userId/link`

لیست کاربران، گزارش عملیاتی و ساخت کد اتصال فقط برای نقش مدیر قابل دسترسی است.
